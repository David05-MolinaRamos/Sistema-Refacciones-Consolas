### 1. Metodología de Desarrollo
- **Metodología elegida:** SCRUM.
  -**Justificación:** SCRUM permite adaptarse a cambios y facilita la coordinación del equipo.Además, funciona bien con herramientas como GitHub y Jira para la gestión de código y tareas.
- **Roles:** Propietario del producto, Organizador, Equipo de Desarrollo y Tester.
- **Ciclos de trabajo:** Sprints de 1 a 2 semanas con reuniones semanales breves.

### 2. Actividades Principales
- **Análisis:** Reunir requisitos mediante entrevistas y escribir historias de usuario.
- **Diseño:** Definir criterios de aceptación, modelado de la base de datos y diseño de interfaz.
-**Desarrollo:** Revisión de código e integración continua.
- **Pruebas:** Pruebas funcionales y de aceptación con usuarios reales (técnico y administrador).
- **Entrega:** Despliegue a producción, capacitación breve y documentación de uso.

### 3. Control de Calidad
- **Prevención de errores:** Definición clara de historias, revisión de código obligatoria por otro desarrollador, backups de la base de datos, validación de datos críticos y pruebas de aceptación.
- **Prácticas a usar:** Iniciar con tests para funciones críticas, usar checklist de despliegue, revisión en pares y registro de incidencias en el tablero de Jira.

### 4. Alcance del Proyecto
- **Objetivo:** Gestionar el inventario de refacciones para consolas de videojuegos dentro de un taller de reparación.
- **Funciones incluidas:** Registrar refacciones, consultar compatibilidad por modelo, visualizar stock disponible y consultar precios aproximados.
- **Control de acceso:** Diferenciado para administrador, técnico y cliente.
- **Exclusiones:** No incluirá pagos en línea, integración con proveedores externos ni comercio electrónico.

### 5. Recursos
-**Recursos Humanos:** Equipo de 4 integrantes organizados en: Líder del proyecto, Desarrolladores, Encargado de pruebas y Encargado de documentación.
- **Recursos Técnicos:** Computadoras personales, repositorio en GitHub, Jira para gestión de tareas, lenguajes de programación y conexión a internet.

### 6. Riesgos y Métricas
- **Riesgos Principales:** Retraso en la integración del equipo, curva de aprendizaje de herramientas, información técnica errónea, conflictos al subir código al repositorio e incumplimiento de fechas.
- **Métricas del Proyecto:**
  - **Tiempo estimado:** 3 semanas.
  - **Costo estimado:** $0 MXN, aprovechando herramientas de código abierto.
  - **Indicadores de avance:** Porcentaje de incidencias terminadas en Jira y cantidad de "commits" en GitHub.

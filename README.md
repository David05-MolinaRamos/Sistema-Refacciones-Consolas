# Sistema-Refacciones-Consolas
Trabajo de la materia de Fundamentos de la Ingeniera de Software de la universidad autónoma de ciudad Juárez (UACJ).  
#Nombre: LosBrosgaming.
#Problemas: La dificultad para gestionar el inventario de refacciones especificas y compatibilidad de componentes para consolas en talleres locales.
#Usuarios: Administradores de stock, Tecnicos de reparacion y Clientes finales.
#Objetivo General: Automatizar el control de existencias por numero de serie para optimizar las reparaciones y ventas.
#1. Metodologia de desarrollo elegida SCRUM.
#i.1 Justificacion: Se eligio SCRUM por ser una metodología ágil que permite la colaboración en equipo a través de ciclos de trabajo (Sprints). Facilita la adaptación a cambios y permite que cada integrante avance en sus tareas de forma independiente usando Jira y GitHub para el control de versiones.
#2. Actividades principales.
#2.1 Análisis: Identificación de las refacciones más comunes y las necesidades de los técnicos.
#2.2 Diseño: Creación del modelo de la base de datos para relacionar piezas con modelos de consolas.
#2.3. Desarrollo: Codificación de la interfaz y la lógica del inventario.
#2.4. Pruebas: Validación de que el sistema descuenta correctamente el stock al realizar una venta.
#2.5. Entrega: Presentación del software funcional al cliente final.
